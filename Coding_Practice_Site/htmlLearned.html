Terms

<html>

Overview

This tag allows html pages to exist. This tag basically tells the program that whatever gets written between it and its closing tag is HTML code. Without it, the program won't know when something is HTML code, and won't register it as such.

Attributes

  • N/A

Notes

  • Needs a closing tag(</html>)

Examples

<head>

Overview

All of the metadata of an HTML document is stored here. Basically, this is where a lot of important information that doesn't get seen by the user goes.

Attributes

  • N/A

Notes

  • Needs a closing tag(</head>)

Examples

N/A

<link>

Overview

The "link" tag is a piece of metadata that connects other documents to the HTML document that the tag is used in. While it has many uses, its most common application is the connect an external stylesheet to an HTML page.

Attributes

  • rel:
  • type:
  • href: This attribute provides the HTML document with the location/name of the document being linked to it, which allows the link to work.

Notes

  • No closing tag needed
  • Goes in the head of the document

Examples

N/A

<title>

Overview

This tag is a piece of metadata that gives the page created by the HTML document a name. While this tag does nothing to the page itself, it does control what is displayed on the page's tab.

Attributes

  • N/A

Notes

  • Needs a closing tag(</title>)
  • Goes in the head of the document
  • Using this is strongly recommended, but not required, to make the page work.

Examples

Look at this page's tab on the browser! It should say, "HTML Learned". That's because that is the title of the page.

<style>

Overview

This tag is a piece of metadata that allows CSS(another programming language) to be used within an HTML document. CSS can be written between it and its closing tag, and the program will understand that it's CSS, and apply it correctly to the webpage.

Attributes

  • N/A

Notes

  • Needs a closing tag(</style>)
  • Goes in the head of the document

Examples

N/A

<body>

Overview

Everything that gets seen by the user of a webpage goes between this tag and its closing tag. In short, this tag contains everything that gets displayed on a page.

Attributes

  • N/A

Notes

  • Needs a closing tag(</body>)
  • Sometimes adds random offsets to webpages. Very annoying.

Examples

Everything seen on ths page! All of it is contained within the body of the HTML code that makes up this page.

<h1-h6>

Overview

h1, h2, h3, h4, h5, and h6 are all header tags that display text. Starting from h1, the header tags get progressively smaller and less important. These header tags are often used to display the titles/labels of webpages/sections of webpages.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</h1>, </h2>, etc)

Examples

This is an h1 tag.

This is an h2 tag.

This is an h3 tag.

This is an h4 tag.

This is an h5 tag.
This is an h6 tag.

<p>

Overview

This is the paragraph tag, which is obviously used to display paragraphs. This tag is most commonly used for large sections of text.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</p>)

Examples

This text is a p tag.

<a>

Overview

This tag is used to create links to other websites, as well as other parts of the website the tag is on. Any piece of text that links to anything else uses this tag.

Attributes

  • href: This attribute provides the HTML document with the location/name of the document being linked to it, which allows the link to work.
  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</a>)

Examples

The "Terms," "Concepts," and "Syntax" texts in the navigation are all a tags that link to other parts of this page. The "Home" text in the footer is an a tag that links to another page.

<img>

Overview

This tag allows images to exist on webpages. It's as simple as that: this tag is how images get displayed.

Attributes

  • src: This attribute provides the HTML document with the location/name/URL of the image being linked to it, which allows the image to be displayed and controls what image it is.
  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Does not need a closing tag
  • Almost always needs its height or width customized, as the default size of it is almost never what is desired

Examples

<div>

Overview

A div tag creates a new box for everything contained between it and its closing tag. This does hardly anything on its own, but it is extremely useful for incorportaing CSS into webpages. Divs are extremely useful for formatting webpages.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</div>)

Examples

The box containing everything about thi tag is a div. So is every other box like this one. This page has several dozen divs within it, and this overall webite contains well over 100.

<span>

Overview

A span tag distingushes any text contanied between it and its closing tag from other text around it, without changing the position of the text. This is very useful for customizing certain sections of text with CSS.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</span>)
  • Nearly useless without an id or a class

Examples

The text that is red is contained within a span.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</table>)

This is where the body of a table(as opposed to its header) is programmed. Generally, the more specific information of a table is stored here.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

<thead>

Overview

This is the headersof a table are programmed. These are generally used to label catgories, and the items in these categories are programmed in the body of the table.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</thead>)

Examples

See the "<table>" section(row 1 is in the head of the table).

<tr>

Overview

This tag contains a row of table boxes between it and its closing tag. The number of rows in a table is equal to the number of times this tag is used within it. Anything written in the first <tr> tag will go in the first row of the table, anything written in the second <tr> tag will go in the second row of the table, and so on.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</tr>)

Examples

See the "<table>" section(rows 1, 2, and 3 are only separate rows because they are contained within 3 different <tr> tags).

<th>

Overview

Each individual header box is programmed between this tag and its closing tag. This tag goes in a table row, and the row can have any number of <th> tags. The first tag in the row will go in column 1, the second tag in the row will go in column 2, and so on.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</th>)
  • Text within this tag is bolded by default
  • This tag goes in the <thead> section, and also within a <tr>

Examples

See the "<table>" section(the two boxes in row 1 are contained within <th> tags).

<td>

Overview

Each individual table body box is programmed between this tag and its closing tag. This tag goes in a table row, and the row can have any number of <td> tags. The first tag in the row will go in column 1, the second tag in the row will go in column 2, and so on.

Attributes

  • (Optional) id: Gives the tag an id.
  • (Optional) class: Gives the tag a class.

Notes

  • Needs a closing tag(</td>)
  • This tag goes in the <tbody> section, and also within a <tr>

Examples

See the "<table>" section(the four boxes in rows 2 and 3 are contained within <td> tags).
