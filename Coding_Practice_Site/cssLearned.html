Terms

color

Overview

This controls the color of the text in the selected region.

Attributes

  • rgb(x, y, z): identifies the desired color in RGB format
  • #uvwxyz: identifies the desired color in hexadecimal format

Notes

  • None

Examples

This text has been colored blue by the color element.

background-color

Overview

This controls the background color of the selected region.

Attributes

  • rgb(x, y, z): identifies the desired color in RGB format
  • #uvwxyz: identifies the desired color in hexadecimal format

Notes

  • This coloirng also applies to the padding of the selected region.

Examples

The background of this box has been colored dark gray.

width

Overview

This controls the width of a selected region.

Attributes

  • xpx/xem/x%: identifies the desired width.

Notes

  • None

Examples

The width of this box has been set to 40%.

height

Overview

This controls the height of a selected region.

Attributes

  • xpx/xem/x%: identifies the desired height.

Notes

  • Setting the height AND width of a box can often result in problems, so only do this when it's absolutely necessary.

Examples

The height of the header of this page has been set to 100px.

position

Overview

This element controls the way in which the selected region is positioned.

Attributes

  • relative: makes it so that any adjustments to the selected region's position are registered as displacement from the default position.
  • absolute: makes it so the selected region ignores all other regions/boxes/sections, and has its position set exclusively by the top/bottom/left/right elements.
  • fixed: this is similar to the absolute setting, except the fixed setting makes it so the selected region NEVER moves on the screen, even when the user scrolls.

Notes

  • This does not set the actual position of the selected region. It only sets the way in which the position is set
  • When a region is set to a fixed or absolute position, it goes to the top right of the page by default

Examples

This box's position has been set to relative, and the footer's position has been set to fixed.

top/bottom/left/right

Overview

These elements contorl the actual position of the selected region. The top element controls how far the selected region is from the top, the bottom element controls how far the selected region is from the bottom, and so on.

Attributes

  • xpx/xem/x%: identifies the desired shift in position.

Notes

  • Top OR bottom can be used, and left OR right can be used, but these pairs can't both be used on the same selection

Examples

The header of the page is set to top: 0px, the navigation of the page is set to top: 100px, and the footer of the page is set to bottom: 0px.

font-size

Overview

This element controls the size of the selected text.

Attributes

  • xpx/xem/x%: identifies the desired text size.

Notes

  • It is strongly suggested to have the font-size of more important text be bigger

Examples

The font-size of this text is set to 20px.

font-family

Overview

This element, predictably, controls the font family of the selected text.

Attributes

  • Georgia, arial, monospace, and helvetica: these are just a few of the many font families that can be used.

Notes

  • Some devices don't register certain font families, so it's recommended to have a secondary font family set in case this happens to some users

Examples

This text has a font-family of georgia.
This text has a font-family of monospace.
This text has a font-family of times new roman.

font-weight

Overview

This element controls the weight of selected text, or in other words, controls how bolded the text is.

Attributes

  • 100-900: these numbers correspond to how bolded the text is, where 100 is the least bold and 900 is the most bold
  • normal: this makes the text have a normal/standard weight
  • bold: this makes the text bold

Notes

  • Most often, using numbers to define the weight of text isn't ideal, as usually only two of those numbers register font weights: 400 and 700
  • If a numberical font weight is used and it doesn't correspond to a weight: the program will just round the number to the nearest number that does have a weight

Examples

This text has a normal weight.
This text has a bold weight.

text-decoration

Overview

This controls the decoration of the selected text, which is most frequently used to simply control whether the selected text is underlined or not.

Attributes

  • none: this makes it so the selected text has no decoration
  • underlined: this makes it so the selected text is underlined

Notes

  • It's normally a good idea to only have links underlined, and nothing else

Examples

This text has no decoration.
This text has an underlined decoration.

font-style

Overview

This element controls the style of the selected text. This is mainly used to italicize selected text.

Attributes

  • normal: does not give the selected text any special style
  • italic: italicizes the selected text

Notes

  • Text has a normal font style by default

Examples

This text has an italicized font style.

text-align

Overview

This element controls how text aligns within its box.

Attributes

  • left: causes the text to align with the left of its box
  • center: causes the text to align with the center of its box
  • right: causes the text to align with the right of its box

Notes

  • Text aligns with the left of its box by default

Examples

This text is aligned with the center of its box, and the text in the above list is aligned with the left of its box.

margin

Overview

This element controls the size of the margin of the selected regions. The margin of a box is the space between its content and its border.

Attributes

  • xpx/xem/x%: identifies the desired margin size.

Notes

  • "-top", "-bottom", "-left", and "-right" can be added to the "margin" term to control specific sides of the margin
  • The margin of a box can be many different values by default, so it's best to always set the margin of boxes

Examples

This box has a margin-bottom value of 10px and a margin-top value of 20px.

border

Overview

This element controls the border of the selected regions.

Attributes

  • xpx/xem/x%: identifies the desired border size.
  • solid, dashed, ridged: a few of the many border styles that can be applied to borders
  • rgb(x, y, z): identifies the desired color in RGB format
  • #uvwxyz: identifies the desired color in hexadecimal format

Notes

  • "-top", "-bottom", "-left", and "-right" can be added to the "border" term to control specific sides of the border
  • The border of a box is rectangular by default

Examples

This box has a solid, gray border, which is 5px thick.

padding

Overview

This element controls the size of the padding of the selected regions. The padding of a box is the space between its border and the outside of the box.

Attributes

  • xpx/xem/x%: identifies the desired padding size.

Notes

  • "-top", "-bottom", "-left", and "-right" can be added to the "padding" term to control specific sides of the padding
  • Padding is generally set to 0px by default

Examples

This text has a padding-bottom value of 20px.

z-index

Overview

This element controls which boxes show up in front of others. Boxes with a higher z-index value than others will show up in front of those others, should they overlap.

Attributes

  • x: identifies the numerical z-index value of the selected box(es).

Notes

  • It's best to customize the z-index value of all boxes, as the program can give different boxes different z-index values by default, which can cause problems when things overlap

Examples

The header of this page has a z-index value of 11, so that it shows up in front of everything.

float

Overview

This element makes a box float, and controls where the box floats to. "Floating" is a different way of positioning boxes, and is usually used to very easily move them to the left or right of the page.

Attributes

  • left: causes the selected box(es) to float left
  • right: causes the selected box(es) to float right

Notes

  • Boxes do not float at all by default

Examples

This box is floating left(as well as every other term box on the left side of the page), and every term box on the right side of the page is floating right.

clear

Overview

This element controls what selected box(es) clear and what they don't. "Clearing" something refers to being positioned below it. Clearing applies almost exclusively to floating boxes.

Attributes

  • none: causes the selected box(es) to clear nothing that floats
  • left: causes the selected box(es) to clear boxes that float left
  • right: causes the selected box(es) to clear boxes that float right
  • both: causes the selected box(es) to clear boxes that float left and right

Notes

  • It is highly recommended to define this element for any box that is meant to go directly under floating boxes, as they sometimes don't do this when this isn't defined

Examples

The "Concepts" div is set to clear both, so it goes below all of the term boxes(including this one).
